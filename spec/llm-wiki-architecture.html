多套实现并存 (6组)

  • • 分块逻辑 ×3: parser.ts / buildIndex / buildIncremental
  • • 实体提取 ×2: LLM / jieba+字典匹配
  • • 检索路径 ×3: 实体注入 / 元信息查询 / RRF语义检索
  • • Query改写 ×2: chat(LLM) / search(jieba仅)
  • • 上下文窗口 ×2: 本地正则 / LLM判断
  • • 索引构建 ×2: 全量 / 增量

高优先级可疑行为

  • • 5.1 增量首次=全量但无DROP → LanceDB重复数据
  • • 5.7 连续3轮追问丢失中间query上下文
  • • 5.9 smartRoute失败时重复调用hybridSearch
  • • 5.11 父文档缓存永不过期，增量更新不感知
  • • 5.12 chunks_meta分片数硬编码
  • • 5.13 search API无会话/多轮能力

核心检索链路 RRF

  • • RRF(d) = Σ 1/(60 + rank_i(d))
  • • 向量检索 top20 + BM25 检索 top20
  • • 实体关键词过滤仅作用于向量侧
  • • 过滤后排名缩进重新编号
  • • qwen3-rerank 重排 Top5 替换分数
  • • 最终分数归一化至 0.05~0.95 区间

状态管理 (隐式依赖)

  • • sessions Map (内存, 30min TTL, setInterval清理)
  • • knownEntitiesCache / parentDocsCache (永不过期)
  • • entityKeywordsCache / entityInvertedIndex (懒加载)
  • • _lancedbConn / _lancedbTable (进程级连接)
  • • 服务重启后全部丢失 · 无持久化
  • • 增量构建后运行时不感知缓存失效
